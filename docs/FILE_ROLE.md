# 📂 FILE_ROLE.md

# 株ラボ ファイル責務一覧

## 🎯 目的

本ドキュメントは、各ファイルの責務（Responsibility）を明確にし、
実装場所の統一・保守性向上・AI開発効率向上を目的とする。

---

# 基本ルール

## 単一責務

1ファイルにつき1つの責務を持つこと。

責務を超える実装は原則禁止とする。

---

## UIとロジックを分離する

UIは画面表示のみを担当する。

計算・判定・分析は別モジュールで実装する。

---

## データ取得を一元管理する

株価取得・優待取得などは
専用モジュールのみが担当する。

# 実装判断フロー

新機能を追加する場合は、以下の順序で判断する。

1. 既存ファイルで責務に合致するものがあるか
2. 責務に合わない場合は新規ファイルを作成する
3. app.pyにはUI以外を書かない
4. データ取得はstock_data.pyへ集約する
5. AI分析はrecommend.pyへ集約する

---

# AI開発ルール

ChatGPTは新機能を提案する際、
必ずFILE_ROLE.mdに従い実装ファイルを決定する。

Claude①・Claude②も
本ドキュメントを優先して実装箇所を判断する。

責務が曖昧な場合は、
既存ファイルへ追記せず、
新規モジュールの追加を優先する。
